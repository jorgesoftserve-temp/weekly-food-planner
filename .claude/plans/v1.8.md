# v1.8 — Major UI Rework & UI/UX Agent Structure (planned)

> **Status: planned / in progress.** Forward plan-of-record for the visual + UX wave between the
> **[v1.5](./v1.5.md)** enhancement tail and the **[v2](./v2.md)** epic. v1.8 does not touch the
> determinism contract, the engine, or the menu/grocery API surface — it is a presentation-layer
> release plus one new per-user preference (`profiles.accent_color`). Guiding principle: **UX over
> prettier screens** — a simple screen with a simple interaction beats a beautiful one that demands a
> complex interaction.

## Context

The app shipped functionally complete (v1 MVP + v1.5) but visually stock: `--primary` is shadcn
slate, there are no brand colors, no custom fonts, no design documentation, and `next-themes` is
installed yet unwired. It is a *food* planner, so a warm, appetizing, comfy/minimalistic look is a
product feature, not polish (cf. recipe sites kiwilimon / cookpad / recetasNestlé; look-and-feel of
Airbnb / Spotify / Todoist). Alongside the rebrand, v1.8 pays down accumulated UX debt: modals can
co-exist/stack (recipe detail + edit drawer; the menu page renders three dialogs from three
independent booleans), and the list pages are desktop-first tables that scroll horizontally on phones
with no mobile reflow.

## Scope — grouped by theme

### Design system & brand
- **Strawberry-red (`#fb4b4e`) brand rebrand** — token-only rewrite of `apps/web/app/globals.css`
  (light + dark), since `components.json` already uses CSS variables. Primary tuned to ~`359 70% 55%`
  for AA white-text contrast; literal `#fb4b4e` reserved for large text / borders / gradient stops.
- **Destructive de-collision** — because the brand is red, destructive becomes deep crimson
  (`0 72% 41%`) + always icon/verb-labeled, so "delete" never reads as "primary".
- **Subtle gradients** — small 8–12% (light) / 4–6% (dark) tints over white/dark surfaces; hero wash,
  card edge, selected-row, empty-state. No heavy full-bleed gradients.
- **Typography** — `next/font/google`: Inter (body) + a serif for headings (recipe-magazine warmth).
- **Docs** — `docs/design/color-palette.md` (base) + `docs/design/user-accent-colors.md` (accents) +
  `docs/design/v1.8-ui-mockups.md` (per-screen wireframes).

### Per-user accent color (Drive / Monday style)
- New `profiles` table keyed to `auth.users(id)` with an `accent_color` enum, self-only RLS, and a
  signup-bootstrap insert (precedent: `sys_create_workspace_on_signup`). Module + React Query hooks +
  types regen.
- Curated safe accent set (excludes too-light pastel-petal): `strawberry` (default), `moss`, `teal`,
  `amber`, `ocean`, `plum`. Applied SSR via `data-accent` on the shell (no FOUC), with optimistic
  client preview. Recolors a constrained surface set (active nav, focus ring, selected chips, links,
  avatar, header wash) — never primary CTAs or destructive.
- **Dark mode** wired (`next-themes`): light / dark / system toggle; every token, gradient, and accent
  gets a dark-tuned variant.

### UX rework (system-wide)
- **Single-overlay rule** — `useExclusiveOverlay` (or per-feature discriminated-union state) so a
  feature has at most one overlay open. Fixes recipe detail+edit coexistence, the menu page's
  three-boolean dialogs, and the IngredientPicker→CreateIngredientDialog stack.
- **Cross-device** — tables get `md:hidden` card-list variants (reusing the menu page's split),
  dialogs become `Sheet side="bottom"` under `md` with `max-h-[85dvh] overflow-y-auto`, offcanvas
  sidebar + header workspace switcher on mobile, touch targets ≥44px, `dvh` + safe-area insets.
  Verified at 390 / 820 / 1440px (iPhone / iPad / Mac).

### Agentic infrastructure
- New build-capable **`design-system-architect`** agent owns the token layer, Tailwind theme, gradient
  utilities, fonts, and the palette docs as source of truth; `ui-component-builder` defers to it on
  tokens; contrast is co-signed by `accessibility-auditor`. Registered in `CLAUDE.md`,
  `docs/agentic/agents.md`, and a `docs/agentic/changelog/` entry.

## Key surfaces (where it lives)
- **Tokens / theme** — `apps/web/app/globals.css`, `apps/web/tailwind.config.ts`,
  `apps/web/app/layout.tsx` (fonts + `next-themes`), `apps/web/app/(app)/layout.tsx` (SSR accent).
- **Accent persistence** — `packages/supabase/supabase/migrations/` (enum + `profiles` + signup fn),
  `packages/supabase/src/module/profiles.ts` (+ `.react.ts`), generated types.
- **Accent UI** — `apps/web/components/app-shell/accent-provider.tsx`,
  `apps/web/app/(app)/settings/_components/accent-picker.tsx`.
- **UX rework** — `apps/web/hooks/use-exclusive-overlay.ts`, `recipes/page.tsx`, `menu/page.tsx`,
  `components/forms/ingredient-picker.tsx`, the three list pages' `_components/`, `app-shell/*`.

## Invariants preserved
The engine, `inputs_hash` / `accepted_seed` boundary, and the menu/grocery route contracts are
untouched. The only new server surface is `profiles` read/update (self-scoped). Accent + theme are
presentation-only; soft-delete, RLS, and the three-Supabase-client rules are unchanged.

## Verification
- `pnpm typecheck && pnpm test`; migration applied locally + `pnpm run gen:types`; `profiles` RLS
  confirmed self-only via the `supabase-local` MCP `query`.
- Accent end-to-end: change in Settings → optimistic preview → persists across reload → follows the
  user into another workspace → correct in light + dark.
- Modal de-dup: recipes never show detail+edit together; menu renders one dialog at a time; ingredient
  create never stacks over the picker popover.
- Responsive at 390 / 820 / 1440px: card lists on phone, bottom-sheet dialogs, offcanvas sidebar, no
  horizontal scroll, ≥44px targets.
- `node scripts/verify-flow.mjs` green; `accessibility-auditor` confirms AA on primary, destructive,
  and every accent in both modes.

## Deliberately out of scope (carried forward)
- PDF export (still post-MVP; layouts stay PDF-ready).
- Per-workspace (vs per-user) theming — v1.8 is per-account only.
- Engine / multi-week / inventory work — remains the **[v2](./v2.md)** epic.

## Phase 2 — Mock-first redesign to a distinct, cleaner system (planned)

Phase 1 rebranded the app but it still *reads* as default shadcn (medium radii, hairline borders,
flat surfaces). Phase 2 designs a distinct, easier-to-use look — **Airbnb-warm, card-forward,
soft-rounded "cozy"** — and reviews it as mocks **before** restyling the live components. Decisions:
in-repo mocks + Playwright MCP now + Figma MCP (dormant) secondary; Airbnb-warm anchor; soft-rounded
cozy shape; **dark-mode toggle ships real now, the shadcn restyle is Phase 3 (post-approval).**

### Shipped in Phase 2
- **Header light/dark/system toggle** — `apps/web/components/app-shell/theme-toggle.tsx`, wired into
  `app-header.tsx` (Phase 1 had it only in Settings).
- **`/design-lab` review route** — `apps/web/app/(design)/design-lab/` (own gated layout,
  `NODE_ENV !== 'production'`, unauthenticated, mock data). Screen switcher + local light/dark preview
  across Dashboard / Recipes / Recipe detail / Menu / Grocery mocks. The **cozy skin is scoped** to
  `[data-skin="cozy"]` (`design-lab.css`) so the live app is untouched.
- **`docs/design/cozy-restyle-spec.md`** — the Phase-3 contract: cozy tokens (radius `1rem`, pill
  radius, soft warm shadow scale, roomier padding) + per-primitive deltas, demonstrated live in the lab.
- **Design tooling MCP** — `playwright` (browser drive/screenshot, no auth) + `figma` (Framelink,
  dormant until `FIGMA_API_KEY`) in `.mcp.json`; registered in CLAUDE.md + `docs/agentic/mcp-servers.md`
  + changelog `2026-06-08_playwright-figma-mcp.md`.
