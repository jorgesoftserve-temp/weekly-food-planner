## What v4.2 is

The discovery surface that makes the community usable: a **browse page** with filtering/sorting, **variant
grouping**, and a personalized **"My community"** tab.

The user's flow items covered here: **#7** (filter by popularity / most-shared / review ranking) and **#3**
(a "My community" tab showing recipes I shared + recipes I imported — *tab UX to be designed when we get there*).

## Scope
- **Browse + sort/filter** — list live publications with sort modes backed by v4.1's aggregates:
  **popularity** (net upvote score), **most-shared** (`import_count`), **review ranking** (`avg_rating` ×
  `review_count`, e.g. a Bayesian/weighted average so a single 5★ doesn't top a 4.8★ with 200 reviews).
  Plus facet filters reusing recipe metadata (cuisine, dietary tags, meal types, `is_official`).
- **Variant grouping (from v4.0's lineage)** — group results by `lineage_id`: show the canonical/original
  publication with its **variants from other users** as "another small version," each crediting the
  `original_creator_name` and its `variant_author`. This is the surfacing the user asked for ("see in the
  community tab potential variants from other users").
- **"My community" tab** — a personalized view: **recipes I shared** (my live publications) + **recipes I
  imported** (my workspace recipes where `imported_from_recipe_id IS NOT NULL`), with their lineage/credit.
  *Exact tab layout/IA is an open design task for when this slice starts.*
- **Import from the page** — wire the v4.0 import action into each browse result.

## Likely surface (to be detailed)
- Mostly **read-side** — a `GET /api/community/recipes` with `?sort=&filter=&group=lineage` params over the
  v4.0/v4.1 publication + aggregate columns; no new tables. A `GET .../community/mine` (or param) for the
  "My community" tab. Possibly a read-only `community_recipes_search` MCP tool (per the MCP roadmap review)
  once browse loops repeat.
- UI feature folder `apps/web/app/(app)/community/` — browse grid (reuse `RecipeCard`), sort/filter chips
  (reuse the recipes/search facet pattern), variant-group accordion, "My community" tab. Delegate to
  **ui-component-builder**; pre-PR **ux-reviewer** + **accessibility-auditor** passes; if mocked first in
  `/design-lab`, a **design-parity-auditor** check.

## Determinism
Pure read-side aggregation/presentation. Engine-free, no schema beyond what v4.0/v4.1 created, no snapshot
regen.
