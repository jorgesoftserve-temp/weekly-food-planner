# v6 — Deeper / ambient AI (research, gated)

> Roadmap: **[v2](./v2.md)** → **[v3](./v3.md)** → **[v4](./v4.md)** → **[v5](./v5.md)** → **v6** (this).
>
> **Status: research / not committed.** The residue of the old "AI assist POC" roadmap after its
> deployment work moved to **[v2.4](./v2.4.md)** and its import/parse/NL-generation/i18n core became
> **[v3](./v3.md)**. What remains here is the AI that needs **net-new tooling the repo does not have today**
> — vision, speech, an agent runtime. Build none of it until v3 proves the suggest-confirm import UX.

## Context

v3 establishes the pattern: AI parses input into drafts, a human confirms, the engine runs pure on the
confirmed data. v6 extends that pattern into ambient/multimodal surfaces that each require a capability the
codebase lacks today. Each item inherits v3's **one hard rule** — AI suggests inputs a human confirms; the
constraint engine is never in the loop; `accepted_seed` is never touched.

## Candidate items (enumerate, don't commit)

- **Voice / dictation capture** — the *voice* half of the old D2. Transcription via the **Web Speech API**
  (client-side, free, already on phones/laptops); AI's value-add is parsing free-form dictation ("chicken
  sandwich, two slices of bread, tomato, ten minutes") into the structured recipe shape — i.e. the same
  parser v3 builds, fed by speech instead of a template. **Missing:** the speech-capture UI + graceful
  fallback when the browser lacks the API.
- **Shopping-list OCR / receipt import** — photograph a receipt → auto-fill the v2.0 shopping session /
  inventory. **Missing:** a **vision model** integration (none today).
- **Smart leftover → recipe suggestions** — "you have rice + chicken → make…", reading the v2.0 leftovers +
  inventory. Suggestions only; the user picks. **Missing:** a suggestion endpoint + ranking.
- **Ambient chat assistant** — a persistent conversational layer over the `menu` MCP tools
  (`engine_generate_menu`, `workspace_preview_menu`, `workspace_member_constraints`,
  `workspace_recipe_usability`, `workspace_recent_menus`). v3's import is request/response; this is an
  ongoing assistant. **Missing:** an **agent runtime** (none today).

## Missing tooling (the gate)

| Item | Net-new tooling needed |
|---|---|
| Voice capture | Web Speech UI + fallback (parser reused from v3) |
| OCR / receipt import | vision model integration |
| Leftover suggestions | suggestion/ranking endpoint over v2.0 inventory |
| Ambient chat | agent runtime + conversational state |

Plus, across the board: server-side WebSearch/WebFetch grounding for real-world recipe values — also absent
today.

## Recommendation
Defer until v3 ships and the suggest-confirm UX is validated with real users. Then pick the cheapest
high-value item (likely voice capture, since it reuses v3's parser) before the tooling-heavy ones
(vision/OCR, agent runtime).

## Determinism
Every item is suggest-confirm and engine-free by construction. AI fills *drafts*; the engine runs pure on
confirmed data; no snapshot regen. Any item that would touch generation inputs must say so explicitly and
follow v3's overlay + intentional-regen discipline.
