# 2026-06-09 — v1.8 Phase 3 complete: cozy mocks promoted into the live app

## What changed

Every approved `/design-lab` cozy mock has been **promoted into the live app** and its mock retired.
Phase 3 restyles the real screens to the cozy language (large radii, soft layered shadows, pill
controls, image-forward cards, Todoist-density lists) using the Stage-A token foundation already in
`globals.css` + Tailwind — so each screen was an **in-place restyle preserving live functionality**,
not a rebuild. Work landed one screen per increment on `stage`, each typecheck-clean and
lean-audited live (Playwright at 1440 light/dark, 0 console errors) before its mock was deleted.

### Screens promoted (each its own commit on `stage`)
- **Members** — responsive card grid + per-member accent (`workspace_members.accent_color`).
- **Dashboard** — bento overview, hero greeting (day-based summary), stat cards, week preview.
- **Recipes** — image-forward card grid (list) + cozy detail dialog + cozy section-card create form.
- **Weekly menu** — cozy draft band (warning tokens + gradient wash), SlotCards with deterministic
  recipe-icon thumbnails; **Generate menu** dialog with a pill segmented Auto/Custom control.
- **Grocery** — per-list `Table` → Todoist-style divide-y row list inside cozy cards (also fixes the
  mobile horizontal-scroll); freshness/allergen cues kept.
- **Profile → live `/settings`** — new gradient-avatar identity header; pill theme toggles; fixed a
  pre-existing `next-themes` hydration mismatch on the theme radiogroup via a mounted-guard.

### Net-new functionality explicitly NOT promoted (deferred to v1.9)
These mock ideas have **no live screen / no data model** yet, so they are not v1.8 styling and stay
as reviewable mocks in `/design-lab`:
- **Cross-module Search** (`search-mock`) — no live `/search` route yet.
- **Cook mode** (`recipe-cook-mock`) — needs `menu_slots.cooked_at` + a "cooked today" surface.
- Grocery per-item **notes / substitution / Replace**, and per-member dashboard filtering.

### `/design-lab` kept as a permanent harness
The lab route, its layout, control surface, chrome-less `frame`, and shared `mock-data` are **kept**.
Only the individual promoted `*-mock.tsx` files were deleted; the `SCREENS` registry now holds just
the two v1.9 future-feature mocks (search, cook mode). This is the reviewable surface for future
versions' mocks before promotion — it is not torn down.

### Accessibility pass
Ran `accessibility-auditor` over the promoted screens. **Safe fixes applied** (commit `42f8af8`):
grocery freshness/allergen cues moved from dropped SVG `aria-label`s to `sr-only` text inside the
ingredient button (icons now `aria-hidden`, hover Tooltip kept); menu + member dropdown triggers
bumped 28px → 36px with decorative icons `aria-hidden`. **Deferred to a dedicated a11y pass**
(pre-existing, not introduced by the cozy restyle): roving-`tabindex` + arrow-key + `tabpanel`
wiring on the three `role="tablist"` widgets (menu day-picker, generate-dialog Auto/Custom,
recipe-detail sections), and the members nested-interactive card refactor.

## Why

The Stage-A token foundation meant the rebrand cascaded automatically; Phase 3 is the deliberate
per-screen polish the v1.8 plan (Part E3 / Part F) called for. Promoting in-place — rather than
swapping in the mock components — keeps every wired affordance (draft/accept, replace/add slot,
shop-for subset, export, overlay-dedup) intact while only the visual language changes. Holding
net-new features back keeps v1.8 a pure UI/UX release with no schema or API contract change.
