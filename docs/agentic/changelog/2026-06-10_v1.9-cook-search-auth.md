# 2026-06-10 ‚Äî v1.9: Cook mode, global Search, cozy Auth

## What changed

v1.9 is the "mini" release on top of `stage` (the v1.8 cozy redesign). Unlike v1.8 it ships the first
**new product features** since the MVP and touches the schema, a new route, and the PRDs. Everything
stays on `stage`; **v1.8 + v1.9 merge to `main` together only after v1.9 is done and tested** (user
directive). Guiding principle carried over: **UX over prettier screens**.

### Part A ‚Äî Cook mode (live)
Promotes the `recipe-cook` mock onto live data.
- **Schema** (`5edf46c`): `menu_slots.cooked_at timestamptz null` + `cooked_by uuid null` (FK ‚Üí
  `workspace_members`, ON DELETE SET NULL), plus a **row-scoped RLS UPDATE policy**
  `menu_slots_cook_mode_update` ‚Äî authenticated workspace members, **accepted menus only**
  (`accepted_at IS NOT NULL`, membership via `fn_user_workspace_role`). These are the only
  member-writable columns on the otherwise service-role-managed table.
- **Route + hook** (`a0a3c47`): `POST ‚Ä¶/menus/[menuId]/slots/[slotId]/cook` (`{cooked}`) sets
  `cooked_at`/`cooked_by` server-side via the caller-context client; the route enforces the column
  restriction (Postgres RLS can't). `useMarkSlotCooked` invalidates the active-menu query.
- **UI**: a full-screen **Cook Sheet** opened from a filled menu SlotCard ‚Äî checkable ingredients +
  numbered steps (ephemeral), sticky "Mark as cooked" / "Mark as not cooked". `MenuView` shows a
  success ring + "Cooked" badge and flips Cook‚ÜíView. Dashboard gains a **"Meals cooked" (N of M)**
  stat over the active menu.
- **Determinism**: `cooked_at`/`cooked_by` are runtime-only and never enter the engine inputs hash ‚Äî
  safe by construction (the snapshot loader never selects `menu_slots`).

### Part B ‚Äî Global search (live, recipes-first) ‚Äî `1bfda5f`
New **`/search`** route promoting the `search` mock. A segmented module bar (Recipes **active**;
menu/grocery/members as disabled **"Soon"** tabs) + recipe facets (keyword + meal + difficulty +
cuisine + dietary, cuisine/dietary derived from the loaded recipes), **filtered client-side over the
existing recipes query** ‚Äî no new endpoint or cross-entity layer. Results reuse `RecipeCard` + the
recipes detail/edit/delete overlays. Wired into the sidebar nav, header breadcrumb, and middleware
protected prefixes. Topbar instant search + cross-module search remain **deferred**.

### Part C ‚Äî Cozy auth
- **Mocks first** (`9420167`): `login` / `signup` / `confirm-email` added to the `/design-lab`
  `SCREENS` registry (kept as the review surface, since middleware redirects authed users away from
  the live login/signup/verify routes).
- **Live promotion** (`9b0c0ed`): the whole `(auth)` surface restyled onto the cozy tokens ‚Äî
  **logic untouched** (`signInWithPassword`, `signUp` + verify, resend, `resetPasswordForEmail`,
  `updateUser`, sanitized `next` redirect). Shared `_components/auth-ui.ts` (rounded-xl fields, pill
  CTAs, accent-strong links, success/warning token notes replacing hardcoded emerald/amber).
- **Layout rework per design feedback**: the `(auth)/layout.tsx` is now a **flat two-surface**
  responsive shell ‚Äî **full-bleed `bg-card` on phone** (no card chrome/margin), a contained
  **two-pane card** (brand panel + form, single `bg-card` with a divider) on a `bg-muted` surround
  that grows `max-w-2xl` (tablet) ‚Üí `max-w-4xl` (desktop). No gradient, so light and dark each read
  as exactly **two colors**. Live-audited at 390/820/1440 √ó light/dark (0 console errors).

### Part D ‚Äî PRD refresh
Reconciled PRD‚Üîcode drift surfaced by `prd-aligner`:
- **PRODUCT_PRD** ¬ß13 (Cook mode) and ¬ß14 (Search) re-versioned **(v1.8)‚Üí(v1.9)** and rewritten to the
  shipped behaviour: Cook mode opens from a menu slot (not recipe detail), "Mark as cooked" is always
  available (not gated on checking every step), dashboard stat is "Meals cooked (N of M, active
  menu)" (not "today"); Search documented as the shipped recipes-first `/search` route with the
  topbar/cross-module tiers deferred.
- **DATABASE_PRD** ¬ß6.12/¬ß8 ‚Äî `cooked_at`/`cooked_by` re-versioned to v1.9; write-path corrected to
  the row-scoped policy + route column discipline (no `sys_set_slot_cooked` RPC).
- **ARCHITECTURE_PRD** ‚Äî `‚Ä¶/slots/:slotId/cook` POST (was a non-existent `/cooked` PATCH); the
  `/api/workspaces/:id/search` endpoint marked **deferred** (search is client-side in v1.9).
