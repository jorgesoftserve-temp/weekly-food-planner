## Global search  вҡ  proposed functionality (two-tier)
Cross-module search, in two tiers so the common case is one keystroke and power users still get
precision:
1. **Simplified вҖ” topbar instant search** (cf. Figma's quick search). A keyword field in the shell
   header; typing shows **grouped results inline** (Recipes / People / вҖҰ), each a thumbnail + name +
   meta, with a **"See all results"** footer that hands off to tier 2. Always available, every screen.
   Demonstrated in `/design-lab` via `topbar-search.tsx` (lightly interactive вҖ” filters mock data).
   - вң… Results render **only once the user types** (empty = a short hint, no dump of everything).
   - вң… **Mobile-first**: under `sm` the header shows a search *icon* that opens a full-width top
     sheet with a large input вҖ” not a shrunken desktop pill (the iPhone/iPad fix).
2. **Advanced вҖ” full Search screen** (cf. Airbnb's segmented query builder). One bar split into
   dropdown segments: **which module** (Recipes / Weekly menu / Grocery / Members) + **per-module
   filters** (recipes: meal, cuisine, difficulty, dietary) + **keyword**. For fine-tuning a query.
   Demonstrated in `/design-lab` Search mock (`search-mock.tsx`), shown open with the module picker
   expanded + results preview.

Cross-module search is new functionality вҖ” flagged for review, mock only.

## Recipe вҖ” Cook mode  вҡ  proposed product addition (not in any PRD yet)
A distinct **cooking checklist** view, opened from a slot in the weekly menu вҖ” what the user reads
*while cooking*. Unlike the details view, **both ingredients AND instructions are checkable** so they
can track progress hands-on (with a "N of M checked" header). Demonstrated in `/design-lab` Cook-mode
mock (`recipe-cook-mock.tsx`) вҖ” now **fully interactive**: checking off **every instruction** reveals
an enabled **"Mark as cooked"** button that completes the dish and returns to the menu. **Worth adding
to PRODUCT_PRD** (a recipe has a read view *and* a cook view); no engine/schema impact вҖ” it's a
presentation of existing recipe data. Flagged here for review; not built into the live app.
